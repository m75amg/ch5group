# 1. 회사 개요

## 1-1. 채널5코리아는 어떤 회사인가

채널5코리아는 e4ds.com을 운영하는 **B2B 기술 미디어 플랫폼 기업**입니다. 전기전자·반도체·임베디드·AI·SDV 분야의 기술 흐름을 해석하고, 이를 뉴스·웨비나·뉴스레터·컨퍼런스·기술 콘테스트·교육 지원 사업으로 연결합니다.

채널5코리아의 본질은 단순 광고대행사가 아니라, **한국에서 가장 도달하기 어려운 사람들 — 전기전자·반도체 엔지니어 — 에게 닿는 통로**를 운영하는 회사입니다. 글로벌 기술 기업들이 한국 엔지니어 시장과 소통하기 위해 채널5코리아의 미디어 자산과 운영 역량을 활용합니다.

## 1-2. 핵심 자산

| 자산 | 설명 |
|---|---|
| **e4ds.com** | 16년 이상 운영된 전기전자·반도체 기술 미디어. 뉴스, 인터뷰, 기술 분석, 웨비나 리뷰 등 누적 콘텐츠 다수. |
| **엔지니어 독자 기반** | 반도체, 임베디드, 자동차 전장, 산업 자동화 분야의 R&D 엔지니어와 기술 의사결정자 중심 독자. |
| **뉴스레터 구독자** | 구독 동의 기반의 검증된 기술 독자 메일링 리스트. |
| **웨비나 운영 경험** | 글로벌 기술 기업과의 웨비나 기획·모집·운영 다수 진행. |
| **컨퍼런스 기획 역량** | Physical AI Conference, e4ds Tech Day, Analog Day 등 매년 산업 아젠다를 만드는 행사 운영. |
| **개발자 콘테스트** | Industrial Edge AI Solution Challenge 등 엔지니어 참여형 캠페인 운영. |
| **글로벌 파트너 네트워크** | 글로벌 반도체·전자부품·계측·임베디드 기업과의 장기 협업. |
| **운영 플랫폼** | e4ds.com, e4ds Webinar, Make.e4ds, KMAP 등 다수의 자체 플랫폼. |

## 1-3. 사업 영역

채널5코리아의 매출 구조는 다음 6개 사업 영역으로 구성됩니다.

1. **배너 광고** — e4ds.com 내 브랜드·캠페인 노출
2. **뉴스레터 마케팅** — 구독자 대상 콘텐츠·캠페인 발송
3. **웨비나** — 온라인 기술 세미나 기획·운영
4. **컨퍼런스** — 대형 산업 행사 기획·운영·스폰서십
5. **기술 콘테스트** — 엔지니어 참여형 챌린지 운영
6. **교육 지원 사업** — 오프라인 교육과 ESG 활동 연계

이 6개 영역은 **독립된 상품이 아니라, 동일한 독자 자산 위에서 운영되는 다양한 커뮤니케이션 형태**입니다. 이 점이 사이트 메시지의 핵심입니다.

# 2. 프로젝트 배경

## 2-1. 왜 별도 사이트가 필요한가

채널5코리아는 현재 e4ds.com을 운영하고 있지만, **e4ds.com과 회사 사이트는 본질적으로 다른 목적**을 가집니다.

| 구분 | e4ds.com | 채널5코리아 회사 사이트 (이번 프로젝트) |
|---|---|---|
| **방문 목적** | 콘텐츠 소비 | 회사 검토 |
| **방문자** | 엔지니어, 기술 독자 | 마케팅 담당자, 사업개발, 채용 후보자 |
| **콘텐츠** | 뉴스, 기사, 웨비나 | 회사 소개, 서비스, 사례, 파트너 |
| **톤** | 기술 미디어 | B2B 코퍼레이트 |
| **전환 목표** | 회원가입, 웨비나 신청 | 서비스 제안 문의 |

두 사이트를 분리하는 이유:
- **사용자 의도가 완전히 다릅니다.** 엔지니어가 뉴스를 보러 오는 사이트에서 회사 정체성을 효과적으로 전달하기 어렵습니다.
- **B2B 영업의 신뢰 확보용 페이지가 필요합니다.** 글로벌 기업 마케터가 본사에 채널5코리아를 추천할 때 보여줄 사이트가 필요합니다.
- **e4ds.com의 콘텐츠 톤과 회사 소개의 비즈니스 톤이 한 사이트에서 공존하기 어렵습니다.**

## 2-2. 현재 풀고자 하는 문제

현재 채널5코리아는 다음과 같은 상황입니다.

1. **회사 정체성을 명확히 보여줄 단일 페이지가 없습니다.** e4ds.com 안의 "회사 소개" 페이지는 미디어 중심으로 작성되어 있어 사업 전체를 보여주기 어렵습니다.
2. **글로벌 본사 보고용 영문 페이지가 부족합니다.** 한국지사 마케터가 본사에 채널5코리아를 소개하려면 영문 자료가 필요합니다.
3. **서비스별 문의 경로가 분산되어 있습니다.** 웨비나, 컨퍼런스, 콘테스트 문의가 서로 다른 페이지나 이메일로 들어옵니다.
4. **미디어킷 다운로드 경로가 명확하지 않습니다.** 광고주가 자료를 받기까지 단계가 많습니다.

이 프로젝트는 위 4가지 문제를 해결하는 단일 솔루션입니다.

# 3. 프로젝트 목적

## 3-1. 비즈니스 목적 (Why)

이 사이트는 다음 3가지를 달성해야 합니다.

### 목적 1. 회사 정체성의 명확한 전달
"채널5코리아는 광고대행사인가? 미디어 회사인가? 행사 회사인가?" — 이 질문에 대해 한 문장으로 답할 수 있어야 합니다.

**답:** *기술 기업과 엔지니어를 연결하는 전기전자 전문 미디어 플랫폼*

### 목적 2. 서비스 문의의 정형화된 진입
6개 사업 영역(배너, 뉴스레터, 웨비나, 컨퍼런스, 콘테스트, 교육)에 대해 **관심 서비스를 선택해서 문의**할 수 있는 단일 경로를 제공합니다.

### 목적 3. 글로벌 본사 보고용 영문 자산
한국지사 마케터가 글로벌 본사에 채널5코리아를 추천할 때 사용할 수 있는 **영문 사이트**를 제공합니다.

## 3-2. 사용자 목적 (What for whom)

각 사용자 유형이 사이트에서 달성해야 할 목적:

| 사용자 유형 | 사이트 방문 이유 | 달성해야 할 목적 |
|---|---|---|
| 글로벌 반도체·전자 기업 마케팅 매니저 | 미디어 파트너 검토 | 30초 안에 채널5의 미디어 자산 규모와 차별점 파악 |

| 사업개발/이벤트 매니저 | 컨퍼런스 스폰서십 검토 | 행사 규모, 청중 구성, 스폰서십 옵션 확인 |
| 교육기관/협회 협업 후보 | 협업 가능성 검토 | 교육 사업 운영 경험 + 협업 사례 확인 |
| 채용 후보자 | 회사 분위기·비전 확인 | 회사 정체성, 비전, 채용 페이지 (2차) |

# 5. 경쟁 환경 & 차별화

## 5-1. 시장 위치

채널5코리아가 속한 시장은 **한국의 B2B 기술 미디어 시장**입니다. 주요 경쟁사는 다음과 같습니다.

| 경쟁사 유형 | 예시 | 채널5와의 차이점 |
|---|---|---|
| 전기전자 전문 미디어 | EE Times Korea, 디일렉, hellot, etnews | 채널5는 미디어 + 행사 + 콘테스트 + 교육의 통합 플랫폼 |
| 일반 IT/산업 미디어 | 전자신문, ZDNet, IT조선 | 채널5는 엔지니어 직군 도달 정확도 우위 |
| 종합 광고대행사 | (수많음) | 채널5는 직접 운영하는 미디어와 독자 자산 보유 |
| 컨퍼런스 전문 기획사 | (개별 회사들) | 채널5는 미디어와 결합된 통합 캠페인 가능 |

## 5-2. 채널5코리아의 차별점

회사 사이트는 다음 4가지 차별점을 명확히 전달해야 합니다.

### 차별점 1. 미디어 + 행사 + 교육의 통합
대부분의 경쟁사가 한두 영역에 집중하는 반면, 채널5코리아는 6개 영역을 **동일한 독자 자산 위에서 통합 운영**합니다. 이 부분이 사이트의 가장 강력한 영업 메시지입니다.

### 차별점 2. 16년의 누적
e4ds.com은 단순 운영 기간 16년 이상의 역사를 가집니다. 새 미디어가 따라잡기 어려운 자산입니다.

### 차별점 3. 엔지니어 직군 도달 정확도
단순 트래픽이 아닌 직군의 정확도. 반도체, 임베디드, 자동차 전장 R&D 엔지니어 비중이 높습니다.

### 차별점 4. 글로벌 기업과의 장기 협업
일회성 광고가 아닌 장기적 기술 커뮤니케이션 협업 이력. 글로벌 본사에 추천 가능한 신뢰도.

## 5-3. 차별화 전달 방식

사이트 곳곳에서 차별점을 다음과 같이 노출합니다.

| 차별점 | 노출 위치 |
|---|---|

| 16년 누적 | Home Hero 지표, About 정체성, Media Platform 인트로 |
| 직군 정확도 | Home Media Snapshot, Media Platform Audience |
| 글로벌 협업 | Partners 페이지 전체, Home Partners 섹션 |

# 10. 기술 방향 (요약)

상세 스택은 `TECH_STACK.md`를 참조합니다. 여기서는 의사결정 배경만 정리합니다.

## 10-1. 왜 Next.js인가

- SEO가 핵심 (검색 유입 중요)
- 한/영 다국어 운영 필요
- 정적 빌드와 SSR 모두 지원
- Vercel과 통합되어 배포·CDN 단순
- TypeScript 친화적

## 10-2. 왜 Tailwind + shadcn/ui인가

- 디자인 토큰 적용 일관성
- 컴포넌트 단위 재사용 용이
- 작은 번들 사이즈
- Claude Code가 가장 잘 다루는 스택

## 10-3. 왜 영문을 1차에 포함하는가

회사 사이트의 핵심 사용자(Persona 1, 3)가 글로벌 기업 한국지사 담당자입니다. 영문이 없으면 본사 보고가 불가능하고, 본사 보고가 안 되면 영업 의사결정이 늦어집니다. 영문은 부가 기능이 아닌 1차 필수 요소입니다.
